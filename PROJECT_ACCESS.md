# Права доступа к проектам в Eggent

## Обзор

Система прав доступа позволяет ограничить доступ пользователей Telegram к определенным проектам в Eggent.

## Файл конфигурации

**Расположение:** `~/.eggent/data/settings/project-access.json`

Этот файл сохраняется при обновлениях Eggent и монтируется в Docker контейнер.

## Правила доступа

1. **Admin пользователи** (поля `adminUserIds`) имеют доступ ко всем проектам
2. **Проектные права** - каждый проект имеет свой список разрешенных пользователей
3. Если пользователь не в `adminUserIds` И не в `allowedUserIds` проекта → доступа нет

## ⚡ Автоматическое добавление новых пользователей

**ВАЖНО:** Если вы добавите пользователя через Eggent UI (настройки Telegram), система **автоматически** добавит его в проект **"Прочее"** при первом сообщении.

Это означает, что:
- ✅ Пользователь сможет работать с ботом сразу после добавления через UI
- ✅ Не нужно редактировать `project-access.json` вручную
- ✅ Пользователь получит доступ к проекту "Прочее" (по умолчанию для новых пользователей)

Если нужно дать доступ к другим проектам - отредактируйте `project-access.json` вручную.

## Обратная совместимость

Если файл `project-access.json` не существует, система работает в старом режиме:
- Все пользователи из `allowedUserIds` в `telegram-integration.json` имеют доступ ко всем проектам

## Управление правами

### Через UI (рекомендуется для новых пользователей)

1. Откройте Eggent UI → Настройки → Telegram
2. Добавьте новый user_id в поле "Allowed User IDs"
3. Сохраните изменения
4. Пользователь автоматически получит доступ к проекту "Прочее" при первом сообщении

### Вручную (для точной настройки)

Отредактируйте файл `~/.eggent/data/settings/project-access.json`:

```json
{
  "projectAccess": {
    "c646d643": {
      "allowedUserIds": [
        "476496854",
        "НОВЫЙ_USER_ID"
      ]
    }
  }
}
```

### Сделать пользователя админом

```json
{
  "adminUserIds": [
    "476496854",
    "НОВЫЙ_ADMIN_USER_ID"
  ]
}
```

## Получение Telegram user_id

1. Отправьте любое сообщение боту в Telegram
2. Бот ответит с вашим user_id, если вы не в списке разрешенных
3. Или используйте команду `/start`

## API для программного управления

Функции реализованы в `src/lib/storage/project-access-store.ts`:

- `getProjectAccessRules()` - чтение правил
- `saveProjectAccessRules()` - сохранение правил
- `canUserAccessProject(userId, projectId)` - проверка доступа
- `getAccessibleProjects(allProjects, userId)` - фильтрация проектов
- `ensureUserHasProjectAccess(allProjects, userId)` - авто-добавление в "Прочее"
- `addUserToProject(userId, projectId)` - добавить пользователя
- `removeUserFromProject(userId, projectId)` - удалить пользователя
- `addAdminUser(userId)` - сделать админом
- `removeAdminUser(userId)` - убрать из админов

## Безопасность

- ✅ Проверка прав доступа происходит на серверной стороне (webhook handler)
- ✅ Пользователь не может обойти ограничения, изменив клиентский код
- ✅ Настройки хранятся в `data/`, которая сохраняется при обновлениях
- ✅ Автоматическое добавление в "Прочее" для новых пользователей через UI
