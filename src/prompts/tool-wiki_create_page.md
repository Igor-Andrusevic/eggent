# Wiki Create Page Tool

Create or update a wiki page to save compiled knowledge.

## Categories

- **sources** — Document summaries (usually auto-created during ingest)
- **entities** — People, organizations, places
- **concepts** — Topics, themes, technical concepts
- **synthesis** — Cross-document analysis, comparisons, insights

## When to Create Pages

- After answering a complex question — save the analysis as a synthesis page
- After comparing information from multiple sources
- When you discover connections between entities or concepts
- To file useful query results that shouldn't disappear into chat history

## Page Naming

Use kebab-case: lowercase letters, numbers, hyphens.
Examples: `john-smith`, `machine-learning`, `quarterly-comparison`

## Content Format

Write in Markdown. Include:
- A clear title (`# Page Title`)
- Substantive content with explanations
- Cross-references using relative links: `[name](../category/name.md)`
- Source references when applicable

## Important

Good answers and analyses should be filed back into the wiki. This way your explorations compound in the knowledge base.
