# 🔄 Руководство по обновлению Eggent с Fork

## 🤖 Автоматические обновления (рекомендуется)

**НОВОЕ:** Теперь Eggent может обновляться **полностью автоматически** с уведомлениями в Telegram!

### Быстрая настройка (2 минуты):

```bash
cd ~/.eggent
bash scripts/setup-update-notifications.sh
```

Скрипт проведёт вас через:
1. ✅ Получение Telegram Chat ID
2. ✅ Тестовую отправку сообщения
3. ✅ Выбор расписания обновлений
4. ✅ Автоматическую настройку cron

### После настройки:

- ✅ **Автоматическая проверка обновлений** по расписанию
- ✅ **Автоматическое применение** через git rebase
- ✅ **Автоматическая пересборка** Docker контейнера
- ✅ **Telegram уведомления** о всех этапах
- ✅ **Обработка конфликтов** с уведомлением

**Подробнее:** См. `docs/AUTO_UPDATE.md`

## 🚀 Пошаговая инструкция обновления

### Шаг 1: Получить изменения из оригинала

```bash
cd ~/.eggent

# Получить обновления
git fetch upstream
```

### Шаг 2: Применить обновления к вашему коду

```bash
# Переключиться на main
git checkout main

# Применить изменения от upstream
git rebase upstream/main
```

**Что делает `git rebase`:**
- Берёт все новые изменения из оригинала
- Применяет их к вашему коду
- Ваши изменения (коммиты) переносятся ВЕРХ новых изменений
- Ваши патчи остаются нетронутыми!

### Шаг 3: Разрешить конфликты (если есть)

```bash
# Если возникнут конфликты:
git status  # Показать файлы с конфликтами

# Редактировать файлы и разрешить конфликты
# После редактирования:
git add <разрешённые файлы>
git rebase --continue
```

### Шаг 4: Отправить изменения в ваш fork

```bash
git push origin main --force-with-lease
```

### Шаг 5: Пересобрать и перезапустить Eggent

```bash
# Проверить, что патчер применится автоматически
npm install

# Пересобрать контейнер
docker compose build --no-cache app

# Перезапустить
docker compose up -d app
```

## 🎯 Типичные сценарии

### Сценарий 1: Регулярное обновление

```bash
cd ~/.eggent
git fetch upstream
git checkout main
git rebase upstream/main
git push origin main --force-with-lease
npm install
docker compose build --no-cache app
docker compose up -d app
```

### Сценарий 2: Отмена изменений

Если что-то пошло не так и хотите откатиться:

```bash
# Вернуть к состоянию до rebase
git reflog
git reset --hard HEAD@{1}

# Или просто откатиться на предыдущий коммит
git reset --hard HEAD~1
```

### Сценарий 3: Сравнить версии

```bash
# Посмотреть разницу между вашим кодом и оригиналом
git diff upstream/main

# Посмотреть лог изменений
git log --oneline upstream/main ^main
```

## ⚠️ Важные замечания

### Ваши файлы, которые не в git:

Все, что в `data/` - это volume, не в git:
- `data/chats/` - история чатов
- `data/settings/` - настройки
- `data/external-sessions/` - сессии

Эти данные **НЕ исчезнут** при обновлении!

### Файлы, которые перезапишутся:

При `git rebase upstream/main` файлы в репозитории перезапишутся на версии из upstream:
- `src/lib/agent/agent.ts` ← БУДЕТ ПЕРЕЗАПИСАН
- `src/app/api/integrations/telegram/route.ts` ← БУДЕТ ПЕРЕЗАПИСАН

**Но ваши коммиты останутся!** Они просто будут перезаписаны на новые версии.

### Как сохранить свои коммиты:

Если upstream изменяет те же файлы, что и вы:

```bash
# Ваши изменения (commits) останутся после rebase:
git log --oneline
# Показывает историю всех ваших коммитов

# Конфликты будут выделены, вы их разрешите вручную
# После разрешения - ваши изменения сохранятся
```

## 🔍 Проверка состояния

### Проверить ahead/behind:

```bash
# Сколько коммитов вы впереди или позади
git rev-list --left-right --count origin/main...upstream/main

# Пример вывода:
# 3 2  # 3 коммита у вас, 2 у оригинала
```

### Сравнить ветки:

```bash
# Графическое сравнение (если есть доступ к ncurses)
git diff --color=always upstream/main

# Список различий
git log --oneline upstream/main ^main
```

## 🎓 Ключевые команды Git

| Команда | Описание |
|---------|----------|
| `git fetch upstream` | Получить обновления из оригинала |
| `git rebase upstream/main` | Применить обновления к вашему коду |
| `git push origin main` | Отправить ваши изменения |
| `git diff upstream/main` | Сравнить ваш код с оригиналом |
| `git log --oneline -5` | Показать последние 5 коммитов |
| `git reflog` | История всех действий |
