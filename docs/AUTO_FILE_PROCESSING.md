# Автоматическая обработка файлов в Eggent

## ✨ Новая функциональность

**Отныне ВСЕ прикрепленные файлы автоматически обрабатываются и становятся доступны для семантического поиска!**

## 🎯 Как это работает

### 1. Прикрепление файла

Когда вы прикрепляете ЛЮБОЙ файл к сообщению в Eggent:

```
Вы: [прикрепляет document.pdf]
Вы: Скажи, о чём этот документ?
```

### 2. Автоматическая обработка (за кулисами)

Eggent автоматически:
- ✅ Сохраняет файл в чат
- ✅ Копирует файл в knowledge base проекта
- ✅ Разбивает файл на chunks (части)
- ✅ Создаёт embeddings (векторные представления)
- ✅ Сохраняет в векторную базу данных

### 3. Семантический поиск

Eggent сразу может искать по содержимому файла:

```
Агент: [использует knowledge_query с "документ"]
       [находит релевантные части]
       [отвечает на вопрос]
```

## 📂 Поддерживаемые форматы

### Текстовые документы:
- `.txt`, `.md` (Markdown)

- `.xlsx`, `.xls` (Excel)
- `.csv`, `.json`
- `.html`, `.xml`
- `.yaml`, `.yml`, `.log`

### Код:
- `.py`, `.js`, `.ts`
- Любые другие текстовые файлы

### Изображения (с OCR):
- `.png`, `.jpg`, `.jpeg`
- `.gif`, `.bmp`, `.webp`

## 🔍 Как агент использует файлы

### АГЕНТ НЕ НУЖДАЕТСЯ В РУЧНОМ ЧТЕНИИ!

**Неправильно:**
```
❌ Агент читает файл вручную через read_text_file
❌ Агент ищет содержимое по ключевым словам
❌ Медленная обработка больших файлов
```

**Правильно:**
```
✅ Агент использует knowledge_query с поисковым запросом
✅ Векторный поиск находит релевантные части
✅ Быстрая обработка файлов любого размера
✅ Семантический поиск (понимает смысл, не только ключевые слова)
```

## 📊 Knowledge Base по проектам

Каждый проект имеет **собственную** knowledge base:

```
Проект "Семья"
├── Файлы из этого проекта
├── Knowledge base этого проекта
└── Поиск только по этим файлам

Проект "Работа"
├── Другие файлы
├── Своя knowledge base
└── Независимый поиск
```

## 🛠️ Технические детали

### Что происходит при загрузке файла:

1. **API:** `POST /api/chat/files`
2. **Сохранение:** Файл сохраняется в `data/chat-files/{chatId}/`
3. **Копирование:** Файл копируется в `data/projects/{projectId}/.meta/knowledge/`
4. **Chunking:** Файл разбивается на части (chunk size: 400 tokens, overlap: 20%)
5. **Embedding:** Каждая часть конвертируется в вектор
6. **Сохранение:** Векторы сохраняются в векторную БД

### Поиск:

1. **API:** Tool `knowledge_query`
2. **Запрос:** Поисковый запрос конвертируется в вектор
3. **Сходство:** Находит наиболее похожие chunks (cosine similarity)
4. **Результат:** Возвращает top-k релевантных частей

## ⚙️ Настройки

### Размер chunk (в настройках проекта):

```json
{
  "memory": {
    "chunkSize": 400,
    "similarityThreshold": 0.7
  }
}
```

- **chunkSize:** Размер куска текста (по умолчанию 400 токенов)
- **similarityThreshold:** Порог схожести (0.0 - 1.0, по умолчанию 0.7)

## 🚨 Ограничения

### Максимальный размер файла:
- Зависит от памяти сервера
- Рекомендуется: до 50MB для PDF, до 10MB для изображений

### Количество файлов:
- Не ограничено (в пределах дискового пространства)

### Поддерживаемые форматы:
- См. список выше
- Другие форматы могут быть добавлены в будущем

## 💾 Хранение файлов

### Где хранятся файлы:

```
~/.eggent/data/
├── chat-files/           # Оригиналы прикрепленных файлов
│   └── {chatId}/
│       └── {filename}
└── projects/             # Knowledge base
    └── {projectId}/
        └── .meta/
            └── knowledge/
                └── {filename}
```

### Векторная база данных:

Хранится отдельно в векторном хранилище ( Depends от векторной БД, используемой в Eggent)

## 🎓 Лучшие практики

### 1. Используйте конкретные поисковые запросы

**Хорошо:**
```
"бюджет на маркетинг в 2024 году"
"сроки выполнения контракта"
"технические требования системы"
```

**Плохо:**
```
"информация"
"данные"
"что есть"
```

### 2. Задавайте уточняющие вопросы

Если первый поиск не дал результатов:
```
Eggent: Я не нашел информации о X в прикрепленных файлах.
       Возможно, она есть под другим названием?
       Хотите, я попробую поискать с другими терминами?
```

### 3. Работа с большими файлами

Для больших файлов (PDF с 100+ страниц):
- Агент найдёт только релевантные части
- Не нужно читать весь файл
- Поиск идёт по содержимому, не по структуре

## 🔧 Troubleshooting

### Файл не находится в поиске

**Возможные причины:**
1. Файл ещё не обработался (подождите несколько секунд)
2. Поисковый запрос не соответствует содержимому
3. Файл в неподдерживаемом формате
4. Порог схожести слишком высокий

**Решение:**
```
1. Проверьте формат файла
2. Используйте более общие поисковые запросы
3. Попробуйте ключевые слова из документа
```

### Ошибки при обработке файла

**Проверьте логи:**
```bash
sudo docker logs --tail=50 eggent-app-1 | grep "knowledge\|import"
```
