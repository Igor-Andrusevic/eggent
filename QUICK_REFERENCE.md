# Quick Reference: Project Access Control

## 🎯 Для новых пользователей (через UI)

### Шаг 1: Получить user_id
Пользователь отправляет любое сообщение боту → получит ответ с user_id

### Шаг 2: Добавить через UI
```
Eggent UI → Dashboard → Messengers → Telegram
Добавить user_id в поле "Allowed User IDs"
Сохранить
```

### Шаг 3: Готово! ✅
Пользователь автоматически получит доступ к проекту "Прочее" при первом сообщении.

---

## 🔧 Для точной настройки (вручную)

### Редактировать файл
```bash
nano ~/.eggent/data/settings/project-access.json
```

### Добавить пользователя в конкретный проект
```json
{
  "projectAccess": {
    "c646d643": {
      "allowedUserIds": [
        "476496854",
        "НОВЫЙ_USER_ID"  // ← добавить сюда
      ]
    }
  }
}
```

### Сделать админом
```json
{
  "adminUserIds": [
    "476496854",
    "НОВЫЙ_ADMIN_ID"  // ← добавить сюда
  ]
}
```

---

## ⚡ Логика автоматического добавления

1. Пользователь добавлен через UI → в `telegram-integration.json`
2. Пользователь отправляет сообщение → проверка `allowedUserIds` ✅
3. Проверка прав доступа → нет ни одного проекта
4. **Автоматически** добавляется в "Прочее" (`0586339d`)
5. Пользователь может работать с ботом ✅

## 🔍 Troubleshooting

### "Нет доступных проектов"
→ Проверьте, что пользователь есть в `project-access.json` или в `adminUserIds`

### Пользователь не может отправлять сообщения
→ Проверьте `telegram-integration.json` → `allowedUserIds`

### После обновления настройки слетели
→ Настройки в `data/settings/` сохраняются. Если слетели - проверьте путь к файлу.
